# UC10 EarnPoints - 포인트를 지급받는다

## Basic Path

1. 시스템은 저장된 운동 기록 데이터(운동 시간, 운동 종류)를 수신한다.
2. 시스템은 포인트 지급 정책(기본 적립, 운동 시간 보너스)을 조회한다.
3. 시스템은 운동 기록 데이터와 정책을 비교하여 지급할 포인트를 산정한다. (A1)
4. 시스템은 오늘 이미 포인트를 지급받은 이력이 있는지 중복 지급 여부를 확인한다. (A2)
5. 시스템은 산정된 포인트를 고객 계정 포인트 잔액에 적립한다.
6. 시스템은 포인트 적립 내역(적립 일시, 적립 사유, 적립 포인트, 누적 포인트)을 저장한다.
7. 시스템은 연속 출석 달성 여부를 확인하여 추가 보너스 포인트 지급 여부를 판단한다. (A3)
8. 시스템은 최종 적립 포인트 정보를 호출한 유스케이스로 반환하며 유스케이스를 종료한다.

## Alternative Path

A1. 운동 시간 기준 미달로 기본 포인트만 지급
  1. 시스템은 운동 시간이 최소 기준(30분) 미만임을 확인한다.
  2. 시스템은 기본 포인트만 산정한다.
  3. 4단계로 돌아간다.

A2. 오늘 이미 포인트 지급 이력 존재
  1. 시스템은 "오늘은 이미 포인트가 적립되었습니다" 메시지를 출력한다.
  2. 추가 포인트 지급 없이 8단계로 돌아간다.

A3. 연속 출석 목표 달성
  1. 시스템은 연속 출석 보너스 포인트를 추가로 산정한다.
  2. 시스템은 보너스 포인트를 계정에 추가 적립한다.
  3. 시스템은 "N일 연속 출석 달성! 보너스 포인트 N점이 추가 적립되었습니다" 알림을 발송한다.
  4. 8단계로 돌아간다.

## Exception

E1. 포인트 정책 조회 실패
  1. 시스템은 포인트 지급을 보류하고 오류 로그를 기록한다.
  2. 시스템은 관리자에게 오류 알림을 발송한다.
  3. 유스케이스를 종료한다.

E2. 포인트 적립 저장 실패
  1. 시스템은 "포인트 적립 중 오류가 발생하였습니다. 고객센터에 문의해주세요" 메시지를 출력한다.
  2. 유스케이스를 종료한다.
