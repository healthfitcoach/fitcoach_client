# UC07 Pay - 결제한다

## Basic Path

1. 시스템은 결제 수단 선택 화면(신용카드, 체크카드, 간편결제, 포인트 사용)을 출력한다.
2. 고객은 사용할 결제 수단을 선택한다.
3. 고객은 결제 수단 정보(카드 번호, 간편결제 인증)를 입력하거나 기존 등록 수단을 선택한다.
4. 시스템은 보유 포인트 잔액을 조회하여 포인트 사용 가능 여부와 잔액을 출력한다. (A1)
5. 고객은 최종 결제 금액을 확인하고 [결제하기] 버튼을 누른다.
6. 시스템은 결제 정보를 결제 게이트웨이(PG사)로 전송하여 승인을 요청한다.
7. 시스템은 결제 게이트웨이로부터 승인 결과를 수신한다. (A2)
8. 시스템은 고객에게 결제 완료 알림(앱 푸시 또는 SMS)을 발송한다.
9. 시스템은 결제 완료 결과를 호출한 구매 유스케이스로 반환하며 유스케이스를 종료한다.

## Alternative Path

A1. 포인트 부분 사용 선택
  1. 시스템은 보유 포인트 잔액과 사용 가능 금액을 출력한다.
  2. 고객은 포인트 사용 금액을 입력한다.
  3. 시스템은 결제 금액에서 사용 포인트 금액을 뺀 잔액을 출력한다.
  4. 4단계로 돌아간다.

A2. 결제 승인 거절
  1. 시스템은 "결제가 승인되지 않았습니다. 카드 정보를 확인하거나 다른 결제 수단을 이용해주세요" 메시지를 출력한다.
  2. 고객은 다른 결제 수단을 선택하거나 결제를 취소한다.
  3. 다른 수단 선택 시 2단계로 돌아간다.
  4. 취소 선택 시 유스케이스를 종료한다.

## Exception

E1. 결제 게이트웨이 연결 실패
  1. 시스템은 "결제 서버와의 연결에 실패하였습니다. 잠시 후 다시 시도해주세요" 오류 메시지를 출력한다.
  2. 유스케이스를 종료한다.

E2. 결제 시간 초과
  1. 시스템은 "결제 요청 시간이 초과되었습니다. 다시 시도해주세요" 오류 메시지를 출력한다.
  2. 유스케이스를 종료한다.
